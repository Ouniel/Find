> 强大的文件搜索工具，支持多种搜索条件和过滤选项，快速定位本地文件系统中的目标文件

File Finder 是一个高效的命令行文件搜索工具，专为开发者和系统管理员设计，提供灵活的搜索条件和实时预览功能，让文件查找变得简单直观。

<br/>

## ✨ 核心功能

- **多维度搜索**：文件名、内容、权限、时间等多条件组合
- **智能过滤**：按文件类型、大小、排除目录精准筛选
- **高性能并发**：多线程搜索加速查找过程
- **可视化输出**：表格化展示结果，支持内容预览
- **日志记录**：详细操作日志便于审计和调试
- **跨平台支持**：完美兼容Windows/Linux系统

<br/>

## 🚀 快速开始

### 安装步骤
```bash
# 克隆仓库
git clone https://github.com/Ouniel/file-finder.git

# 进入项目目录
cd file-finder

# 编译项目
go build -o file-finder

# 安装到系统路径 (Linux/macOS)
sudo mv file-finder /usr/local/bin/

# 验证安装
file-finder -version
```

### 基础使用
```bash
# 在当前目录搜索包含"flag"的文件
file-finder -keyword flag

# 全局搜索可读写文件
file-finder -perm rw -global
```

### 搜索示例
```bash
# 搜索最近修改的配置文件
file-finder -types "conf,ini" -time "2024-03-20" -log

# 查找大尺寸日志文件 (排除临时目录)
file-finder -types "log" -size 10485760 -exclude "tmp,cache" -workers 8
```

## 🛠️ 参数详解

### 搜索参数
| 参数 | 缩写 | 说明 | 示例 |
|------|------|------|------|
| `-keyword` | `-k` | 文件名或内容关键字 | `-k "config"` |
| `-perm` | `-p` | 文件权限(r:读,w:写) | `-p rw` |
| `-time` | `-t` | 修改时间(YYYY-MM-DD) | `-t "2024-01-01"` |
| `-types` | `-ty` | 文件类型扩展名 | `-ty "txt,log"` |
| `-size` | `-s` | 文件大小限制(字节) | `-s 1048576` |
| `-exclude` | `-e` | 排除目录 | `-e "tmp,logs"` |

### 性能参数
| 参数 | 缩写 | 说明 | 示例 |
|------|------|------|------|
| `-global` | `-g` | 全局搜索(所有驱动器) | `-g` |
| `-dir` | `-d` | 指定搜索目录 | `-d "/var/log"` |
| `-depth` | `-dp` | 搜索深度 | `-dp 3` |
| `-workers` | `-w` | 并发线程数 | `-w 8` |
| `-log` | `-l` | 启用日志记录 | `-l` |

## ⚙️ 技术实现

### 搜索流程图
```mermaid
graph TD
    A[启动搜索] --> B{全局搜索?}
    B -->|是| C[遍历所有驱动器]
    B -->|否| D[扫描指定目录]
    C --> E[应用排除目录]
    D --> E
    E --> F[并发文件处理]
    F --> G{匹配条件?}
    G -->|是| H[收集结果]
    G -->|否| I[跳过]
    H --> J[表格化输出]
    I --> F
```

### 性能优化策略
1. **目录预过滤**：跳过排除目录和系统目录
2. **并发池管理**：限制最大goroutine数量
3. **文件类型优先**：先按扩展名快速过滤
4. **大小阈值检查**：提前跳过过大文件
5. **内容搜索优化**：大文件分块读取

## 📌 注意事项

<details>
<summary><b>全局搜索的特殊处理</b></summary>

- **Windows系统**：自动扫描所有可用驱动器(C:, D: 等)
- **Linux/Unix系统**：从根目录(/)开始搜索，可能需要sudo权限
- **默认排除目录**：
  - `$Recycle.Bin`
  - `System Volume Information`
  - `proc`, `sys` (Linux)
  - `tmp`, `.cache`
</details>

<details>
<summary><b>性能优化建议</b></summary>

1. **限制搜索范围**：
   ```bash
   # 指定目录代替全局搜索
   file-finder -k "error" -d "/var/log"
   ```
   
2. **使用深度控制**：
   ```bash
   # 仅搜索3层子目录
   file-finder -k "config" -dp 3
   ```
   
3. **文件类型过滤**：
   ```bash
   # 只搜索配置文件
   file-finder -k "password" -ty "conf,ini,env"
   ```
   
4. **合理设置线程数**：
   ```bash
   # SSD建议8-16线程，HDD建议4-8线程
   file-finder -k "report" -w 12
   ```
</details>

<details>
<summary><b>日志文件说明</b></summary>

- **位置**：程序运行目录下的 `file_finder.log`
- **启用方式**：添加 `-log` 参数
- **日志内容**：
  - 搜索参数配置
  - 已排除目录列表
  - 搜索进度统计
  - 错误和警告信息
- **示例**：
  ```
  [INFO] 2024-03-21 10:30:15 搜索启动
  [PARAM] 关键字: config, 目录: /etc, 深度: 3
  [STAT] 已扫描: 1425 文件, 匹配: 8 文件
  ```
</details>
